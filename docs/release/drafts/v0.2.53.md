# v0.2.53 — WinUI self-contained 폴더 정리 hotfix

## 핵심 변경
- `v0.2.52`에서 WinUI 런타임 문제를 해결하기 위해 self-contained 파일을 모두 포함하면서, `SkyrimDiagWinUI` 폴더 안에 런타임 DLL이 너무 많이 노출되는 문제가 있었습니다.
- `v0.2.53`에서는 `SKSE/Plugins/SkyrimDiagWinUI/SkyrimDiagDumpToolWinUI.exe`를 찾기 쉬운 런처로 유지하고, 실제 self-contained 앱과 런타임 파일은 `SKSE/Plugins/SkyrimDiagWinUI/app/` 아래로 모았습니다.
- Helper 설정의 기존 `DumpToolExe=SkyrimDiagWinUI\SkyrimDiagDumpToolWinUI.exe` 경로는 그대로 유지됩니다.

## WinUI / 사용성
- 사용자가 직접 실행할 파일은 여전히 `SkyrimDiagWinUI/SkyrimDiagDumpToolWinUI.exe`입니다.
- `app/` 폴더는 건드리지 않는 런타임 폴더로 보면 됩니다.
- UI, 분석 흐름, CrashLogger 파서, 후보 산정 로직 변경은 없습니다.

## 엔진 변경
- 작은 Windows 네이티브 런처를 추가했습니다.
- 런처는 받은 인자를 그대로 `app/SkyrimDiagDumpToolWinUI.exe`로 넘기고, 실제 앱이 종료될 때까지 기다린 뒤 같은 종료 코드를 반환합니다.
- Helper의 즉시 종료 감지 로직과 headless fallback 동작이 깨지지 않도록 런처가 자식 프로세스를 기다립니다.

## 빌드 / 운영
- `scripts/package.py`가 WinUI publish output을 `SkyrimDiagWinUI/app/`에 복사하고, CMake가 빌드한 런처 exe를 `SkyrimDiagWinUI/SkyrimDiagDumpToolWinUI.exe`로 복사합니다.
- `scripts/release_contract.py`의 필수 zip 엔트리를 새 app 하위 레이아웃으로 갱신했습니다.
- release gate의 nested-path guard는 `app/` 아래에서도 중복 publish/win-x64/x64 폴더가 들어가지 않도록 확인합니다.

## 주의사항
- `SkyrimDiagWinUI/app/` 폴더 내부 파일을 일부만 삭제하거나 이동하면 WinUI 뷰어가 실행되지 않을 수 있습니다.
- 기존 `v0.2.52`도 런타임 오류 대응은 되어 있지만, 폴더 가독성은 `v0.2.53`에서 개선됩니다.
