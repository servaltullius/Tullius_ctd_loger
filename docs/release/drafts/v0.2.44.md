# v0.2.44 — machine-code capture 보강

## 핵심 변경
- 기본 crash dump와 crash recapture dump가 이제 code segments까지 포함합니다.
- 외부 reverse-engineering/disassembly 도구가 dump 내부에서 기계어를 찾지 못해 실패하던 케이스를 줄이는 목적의 패치입니다.
- 기존 `DumpMode=1` 기본 경로를 유지하면서도, `DumpMode=2 FullMemory`로 바로 올리지 않고 확보할 수 있는 분석 단서를 늘렸습니다.

## 엔진 변경
- Helper:
  - dump profile에 `includeCodeSegments` 플래그 추가
  - `MiniDumpWithCodeSegs`를 dump type 계산에 반영
  - crash 기본 profile / crash recapture profile에서 code segments 요청
- Output / metadata:
  - incident manifest에 `include_code_segments` 기록
  - summary/report에 code segment 포함 여부 노출

## 사용자 영향
- 새로 캡처되는 dump부터 적용됩니다.
- 이미 생성된 예전 dump는 code segments가 없을 수 있으므로, 같은 증상을 다시 분석해야 하면 새 버전으로 재캡처해야 합니다.
- `DumpMode=0` mini 경로는 그대로 경량 우선이며, 이번 개선은 기본 권장 경로인 `DumpMode=1`과 recapture 경로에 집중됩니다.
