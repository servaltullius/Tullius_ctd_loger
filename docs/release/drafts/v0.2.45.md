## 핵심 변경
- EXE / system-victim CTD에서 Crash Logger의 direct fault, first actionable probable frame, same-DLL streak를 더 강하게 반영하도록 분석 엔진을 보강했습니다.
- Crash Logger가 없는 경우에도 Tullius 단독 callstack 단서를 별도 경로로 드러내, 약한 stack-scan 추정과 구분해서 읽을 수 있게 했습니다.
- freeze / hang 분석은 WCT 2회 캡처 기반 consensus와 richer snapshot metadata를 사용해 `deadlock`, `loader stall`, `ambiguous freeze`를 더 보수적으로 구분합니다.

## WinUI / 사용성
- WinUI 공유 텍스트와 텍스트 리포트에 `Crash Logger reading path`, `Next action`, `capture quality`를 직접 표시하도록 정리했습니다.
- `conflicting`, `reference_clue`, `frame-first`, `standalone callstack` 케이스를 실제 공유 텍스트 fixture로 고정해, 사용자가 복사해 보는 문장층 회귀를 줄였습니다.
- freeze summary/report에서도 `support_quality`, `wct_consensus`, `snapshot_consensus_backed` 정보를 직접 읽을 수 있게 했습니다.

## 엔진 변경
- crash dump profile에 `process_thread_data`, `full_memory_info`, `module_headers`, `indirect_memory`, `ignore_inaccessible_memory`를 추가하고 callback-shaped dump bootstrap을 넣었습니다.
- `Crash Logger frame-first` CTD 경로를 중심으로 `frame + first-chance`, `frame + history`, `frame + resource_provider` 보강 신호를 재정렬했습니다.
- freeze snapshot은 `VA_SPACE` / `SECTION_INFORMATION`까지 확장했고, WCT는 2회 캡처 후 `capture_passes`, `cycle_consensus`, `consistent_loading_signal`, `longest_wait_tid_consensus`를 기록합니다.
- `PSS snapshot`은 여전히 opt-in입니다. 기본값은 `EnablePssSnapshotForFreeze=0` 유지입니다.

- 실제 모드팩 환경에서 최신 build 산출물로 갱신 후 운용 확인을 거친 상태입니다.

## 주의사항
- `resource_provider`는 이번 릴리즈에서 바로 상태 승격까지 열지 않았고, score-only 보강으로 제한했습니다.
- legacy freeze / hang 산출물처럼 새 consensus 메타데이터가 없는 샘플은 의도적으로 `ambiguous` 쪽으로 보수 해석됩니다.
- `PSS snapshot` 기반 freeze capture는 아직 opt-in 기능입니다.
