# v0.2.52 — WinUI 런타임 self-contained hotfix

## 핵심 변경
- `SkyrimDiagDumpToolWinUI.exe`가 Windows App Runtime 1.8/MSIX 설치 상태 때문에 시작하지 못하던 배포 문제를 우회하도록 WinUI 뷰어를 self-contained로 전환했습니다.
- `v0.2.52+` 릴리즈 zip에는 WinUI 실행에 필요한 .NET/Windows App SDK 파일이 `SKSE/Plugins/SkyrimDiagWinUI/` 아래에 함께 포함됩니다.
- 이 변경으로 사용자는 WinUI 뷰어 실행을 위해 .NET Desktop Runtime 8 또는 Windows App Runtime 1.8을 따로 설치할 필요가 없습니다.

## WinUI / 사용성
- 뷰어 UI와 분석 흐름은 그대로입니다.
- 설치 실패 안내를 따라 Windows App Runtime 2.x를 설치해도 v0.2.49~v0.2.51 뷰어가 계속 실패할 수 있던 혼선을 줄였습니다.
- zip 크기는 기존보다 커집니다. 이는 self-contained 배포에 필요한 런타임 파일이 같이 들어가기 때문입니다.

## 엔진 변경
- 분석 엔진, CrashLogger 파서, 후보 산정 로직 변경은 없습니다.
- 이번 릴리즈는 WinUI 배포/패키징 hotfix입니다.

## 빌드 / 운영
- `scripts/build-winui.cmd`가 `dotnet build` 대신 `dotnet publish --self-contained true`를 사용합니다.
- WinUI 프로젝트에 `WindowsAppSDKSelfContained=true`를 명시해 Windows App SDK framework package 파일을 publish 출력물에 포함합니다.
- Release gate의 zip 크기 hard limit을 self-contained 배포에 맞춰 100MB로 조정했습니다.

## 주의사항
- 기존 `v0.2.49`~`v0.2.51`의 WinUI 뷰어는 Windows App Runtime 1.8을 별도 설치해야 하는 framework-dependent 배포물이었습니다.
- Windows App Runtime 2.x 설치 로그가 성공해도 1.8 기반 앱 요구사항을 대신 만족한다고 보장할 수 없습니다.
- 이미 설치한 폴더 위에 덮어쓸 때는 `SKSE/Plugins/SkyrimDiagWinUI/` 내부 파일이 일부만 섞이지 않도록 새 zip으로 전체 교체하는 것을 권장합니다.
