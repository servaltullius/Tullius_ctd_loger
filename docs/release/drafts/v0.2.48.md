## 한눈에 보기
- 이번 버전은 **Crash Logger 단서를 Tullius 결론과 더 정확히 맞추는 후속 패치**입니다.
- `CALL STACK ([P]robable / [S]tack scan)` 형식의 Crash Logger 로그를 이제 그대로 읽어들입니다.
- non-system DLL direct fault가 있어도, 독립 근거가 없으면 더 이상 곧바로 `유력 후보 / 높음`으로 단정하지 않습니다.

## 핵심 변경
- Crash Logger parser가 mixed `[P]/[S]` call stack 헤더를 인식하고, 실제 probable row인 `[P]`만 후보 근거로 수집합니다.
- Summary 문장이 `cross_validated / related / reference_clue / conflicting` 상태를 구분해, DLL 내부 fault와 실제 근본 원인을 같은 의미로 섞어 쓰지 않도록 조정했습니다.
- Recommendations도 약한 DLL 단서에는 `root cause로 단정하지 말 것`을 우선 안내하고, mod author 보고 권고는 교차검증된 경우에만 유지합니다.

## 엔진 / 해석 품질
- 실제 사용자 사례처럼 Crash Logger raw log에는 여러 DLL이 보이는데 Tullius가 direct-fault DLL 하나로 과축약하던 케이스를 줄입니다.
- `first_actionable_probable_module`, `top_modules`, probable streak 같은 Crash Logger 보강 단서가 mixed format 로그에서도 다시 살아납니다.
- non-system DLL fault case는 `victim location` 가능성을 더 분명히 드러내, 리포트가 마지막에 맞아 죽은 DLL을 범인처럼 과신하지 않게 합니다.

## 빌드 / 운영
- 이번 변경은 parser 테스트와 summary/recommendation source guard를 함께 확장해 회귀를 고정했습니다.
- 배포 zip은 `Tullius_ctd_loger_v0.2.48.zip` 형식을 사용합니다.

## 주의사항
- 이번 릴리즈는 Crash Logger clue 해석 정렬에 집중한 패치입니다. `address_db` 커버리지나 새로운 게임 버전 함수 DB 추가는 포함하지 않습니다.
- MO2가 닫히는 별도 현상은 Tullius 없이도 재현된 독립 이슈로 확인되어, 이번 릴리즈 범위에 포함하지 않습니다.
