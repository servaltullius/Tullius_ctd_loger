# v0.2.49-rc1 — non-system DLL CTD 과단정 완화

## 핵심 변경
- non-system DLL CTD도 `actionable_candidates` 합의 경로에 들어가도록 분석 엔진을 확장했습니다.
- `Crash Logger frame + 같은 덤프 stack`만으로는 더 이상 독립 교차검증처럼 승격하지 않고, `fault-location cluster`로 더 보수적으로 해석합니다.
- 그래서 `Detection Meter` 같은 raw fault DLL이 계속 보이더라도, 근거가 약한 경우에는 `유력 후보`보다 `현재 fault-location 단서`에 가깝게 읽히도록 바뀝니다.

## WinUI / 사용성
- Triage / report / recommendation에서 weak non-system DLL case를 더 조심스럽게 표현합니다.
- `콜스택 기반 유력 후보`처럼 읽히던 경로를 `fault-location 단서`, `주변 probable DLL 비교` 흐름으로 낮췄습니다.
- 사용자는 raw crash site와 Tullius 해석을 이전보다 덜 혼동하게 됩니다.

## 엔진 변경
- `BuildActionableCandidates`가 이제 non-system DLL CTD도 처리합니다.
- `fault module`과 일치하는 candidate가 `Crash Logger frame + same-dump stack/history` 정도만 가진 경우, low/cautious path로 강등합니다.
- summary/evidence/recommendation이 이 weak candidate 상태를 공통으로 읽도록 정렬했습니다.

## 빌드 / 운영
- 테스트용 prerelease zip은 `Tullius_ctd_loger_v0.2.49-rc1.zip` 형식을 사용합니다.
- GitHub prerelease로 먼저 배포해 실제 CTD 사례에 대한 체감 정확도를 확인할 수 있게 합니다.
- 정식 패치는 이 prerelease 피드백을 반영한 뒤 판단합니다.

## 주의사항
- `Module+Offset`, `추정 모드` 같은 raw field는 여전히 실제 fault-location 정보를 보여줍니다.
- 따라서 weak case에서도 같은 DLL 이름 자체는 계속 보일 수 있습니다. 이번 프리릴리즈의 목적은 `그 이름을 범인 확정처럼 읽히지 않게` 하는 데 있습니다.
- 필요하면 다음 단계에서 `추정 모드` 라벨도 `fault-location inferred mod` 성격으로 더 명확히 바꿀 수 있습니다.
