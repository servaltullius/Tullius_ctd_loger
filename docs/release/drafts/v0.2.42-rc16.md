# v0.2.42-rc16 — 엔진 정확도 강화 + 덤프 intake 개선

## 핵심 변경
- EXE / system-victim CTD 해석을 강화했습니다. `object-ref 중심 + 교차검증 게이트`로 후보를 정리하고, `first-chance` 문맥을 약한 보강 신호로 연결했습니다.
- freeze / ILS 분석을 `deadlock_likely`, `loader_stall_likely`, `freeze_candidate`, `freeze_ambiguous`로 나누고, WCT / blackbox / snapshot 신호를 함께 사용하도록 확장했습니다.
- recapture 정책을 `crash_richer`, `crash_full`, `freeze_snapshot_richer`로 분리해, 애매한 사고를 무조건 full dump로 올리지 않고 상황별로 다시 수집하도록 바꿨습니다.

## WinUI / 사용성
- 시작 화면을 `최근 발견된 덤프`와 `덤프 출력 위치` 중심으로 바꿨습니다. MO2 `overwrite` 또는 `OutputDir` 위치를 기준으로 자동 발견합니다.
- Triage 화면은 `CrashLogger 기준 -> 근거 합의 -> 다음 조치` 순서로 읽히도록 재배치했고, recommendations 영역에 `recapture context`를 표시합니다.
- 행동 우선 후보 이름은 `friendly label`보다 `ESP/ESM 파일명`, `DLL 파일명`을 우선 사용합니다.

## 엔진 변경
- `DumpMode` 단일 3단 구조를 incident-aware capture profile로 확장했습니다.
- freeze/manual capture에는 opt-in `PSS snapshot` 경로를 추가했습니다. 기본값은 여전히 `EnablePssSnapshotForFreeze=0`입니다.
- blackbox에 `module load/unload`, `thread create/exit`, `first-chance exception`을 추가했고, analyzer가 loader stall / freeze / CTD 해석에 이를 소비합니다.
- symbol/runtime health, recapture evaluation, actionable candidate naming policy를 analyzer/output 전반에 반영했습니다.

## 빌드 / 운영
- WSL에서 Windows 빌드를 안정적으로 호출하는 래퍼를 추가했습니다.
- prerelease 노트 템플릿과 `--notes-file` 흐름을 repo에 고정했습니다.
- 메인 CI에서는 flaky한 WinUI headless smoke를 제거하고, 수동 workflow로 분리했습니다.
- 릴리즈 판단 기준은 GitHub Actions가 아니라 로컬 검증 번들 기준으로 정리했습니다.

## 주의사항
- generic Windows `CrashDumps`는 자동 발견 대상이 아닙니다. 툴리우스 출력 위치 기준으로 동작합니다.
- `PSS snapshot` 기반 freeze capture는 아직 opt-in 기능입니다.
- `first-chance`와 blackbox 문맥은 보강 신호이며, 단독으로 후보를 확정하지 않습니다.
