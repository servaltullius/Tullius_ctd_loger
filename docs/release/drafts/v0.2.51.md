# v0.2.51 — CrashLoggerSSE v1.21/v1.22 객체 로그 호환성 보강

## 핵심 변경
- CrashLoggerSSE v1.21/v1.22에서 추가된 객체 introspection 출력에 맞춰 `POSSIBLE RELEVANT OBJECTS` 파서를 보강했습니다.
- `RE::SpellItem`, `RE::EffectSetting`, `RE::BGSLocation`, `RE::NavMesh` 요약 라인에서도 plugin 파일명, FormID, 객체 타입, 표시 이름을 최대한 유지합니다.
- 기존 Crash Logger 형식과 기존 Tullius 로그 분석 결과는 유지하면서, 새 요약 포맷만 fallback으로 처리합니다.

## WinUI / 사용성
- WinUI 화면 구조나 조작 흐름 변경은 없습니다.
- 사용자는 새 CrashLoggerSSE 로그를 넣었을 때 spell/effect/location/navmesh 단서가 빈 객체 정보로 내려가는 대신, 가능한 경우 타입과 이름까지 함께 볼 수 있습니다.

## 엔진 변경
- `RDX: RE::SpellItem "..." [0x...] (SomeMod.esp)` 같은 simplified introspection 라인에서 `RE::...` 타입명을 추출합니다.
- 타입명이 parenthesized pointer 형식이 아닌 경우에도, 표시 이름이 plugin 파일명과 섞이지 않도록 안전하게 이름만 추출합니다.
- 새 CrashLoggerSSE 객체 라인을 대상으로 한 regression test를 추가했습니다.

## 주의사항
- 이번 릴리즈는 CrashLoggerSSE 최신 로그 호환성 보강 중심입니다. 캡처 기본값, UI 레이아웃, 진단 점수 정책은 바꾸지 않았습니다.
- CrashLoggerSSE가 plugin 파일명을 포함하지 않는 순수 mesh/runtime 객체만 출력하는 경우에는 기존처럼 plugin 후보로 승격하지 않습니다.
