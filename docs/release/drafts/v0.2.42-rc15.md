# v0.2.42-rc15 — 후보 식별자 정리 + 로컬 검증 기준 정비

## 핵심 변경
- 행동 우선 후보 이름을 더 실용적으로 바꿨습니다. 이제 `friendly label`보다 `ESP/ESM 파일명`, `DLL 파일명` 같은 실제 식별자가 우선 표시됩니다.
- EXE / system-victim CTD처럼 애매한 케이스에서 `Faction Ranks` 같은 모호한 라벨보다 `FactionRanks.esp`, `paragon-perks.dll`처럼 실제로 찾을 수 있는 이름이 먼저 보입니다.

## WinUI / 사용성
- WinUI 상단 후보 카드와 후보 리스트가 사람 친화 라벨보다 파일 식별자를 우선 사용합니다.
- 보조 라벨은 상세 설명으로만 남겨, 사용자가 MO2/Vortex/파일 탐색기에서 바로 대상을 찾기 쉽게 했습니다.

## 엔진 변경
- actionable candidate naming policy를 backend/output/UI 전반에 공통 적용했습니다.
- 대표 이름 우선순위는 `plugin filename > dll filename > mod folder name > friendly label` 입니다.

## 빌드 / 운영
- 메인 CI에서 flaky한 `WinUI headless interop smoke`를 제거하고, 수동 workflow `.github/workflows/winui-headless-smoke.yml`로 분리했습니다.
- 릴리즈/프리릴리즈 판단 기준을 GitHub Actions가 아니라 로컬 검증 번들로 명시했습니다.
- `AGENTS.md`, `docs/DEVELOPMENT.md`, prerelease 템플릿도 이 기준으로 정리했습니다.

## 주의사항
- GitHub Actions는 이제 선택적 참고용입니다. 릴리즈 판단은 로컬 Linux 테스트, Windows 빌드, 패키징, release gate 기준으로 진행합니다.
- 수동 WinUI headless smoke가 필요하면 GitHub Actions의 `winui-headless-smoke.yml`을 직접 실행해야 합니다.
