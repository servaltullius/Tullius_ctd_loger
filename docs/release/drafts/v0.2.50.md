# v0.2.50 — MO2 provider/slot 진단 정확도 보정

## 핵심 변경
- 활성 MO2 profile의 `modlist.txt`를 읽은 경우, 꺼져 있는 모드가 loose-file provider 후보처럼 표시되지 않도록 보정했습니다.
- ESL / full plugin 슬롯 경고는 활성 플러그인 기준으로 계산되며, full plugin 판정이 불확실한 항목은 고신뢰 슬롯 한계 경고에서 제외됩니다.
- `G:\skyrim project\Tullius_ctd_loger`로 옮긴 현재 repo 위치에서도 릴리즈 게이트와 외부 build tree 테스트가 정상 동작하도록 정리했습니다.

## WinUI / 사용성
- WinUI 화면 구조나 조작 흐름 변경은 없습니다.
- 이번 버전의 체감 변화는 분석 결과에서 비활성 모드나 불확실한 슬롯 정보가 원인 후보처럼 과하게 보이는 상황을 줄이는 데 집중되어 있습니다.

## 엔진 변경
- MO2 provider index가 활성 profile modlist를 읽은 경우 provider 후보를 활성 모드로 제한합니다.
- `esl_count_gte` 조건은 활성 ESL만 세도록 변경했습니다.
- 새 `full_plugin_count_gte` 조건을 추가해, 활성 full plugin 수를 별도로 판정할 수 있게 했습니다.
- `slot_type_known=false`인 항목은 full plugin 슬롯 한계 경고 계산에서 제외됩니다.

## 빌드 / 운영
- `scripts/verify_release_gate.sh`의 기본 repo root가 스크립트 위치 기준으로 계산됩니다. 예전 WSL checkout / Windows mirror 경로를 기본값으로 쓰지 않습니다.
- WSL에서 Windows helper runtime test를 실행하는 문서 예시는 현재 작업 디렉터리를 `wslpath -w "$PWD"`로 변환하도록 정리했습니다.
- repo 밖 build tree에서도 `skydiag_candidate_consensus_tests`가 source root를 찾도록 CTest 환경을 보강했습니다.

## 주의사항
- 이번 릴리즈는 분석 정확도와 릴리즈 운영 경로 보정 중심입니다. 새 UI 기능이나 런타임 캡처 기본값 변경은 없습니다.
- `slot_type_known=false`가 포함된 불완전한 plugin scan에서는 full plugin 슬롯 경고가 더 보수적으로 동작합니다.
