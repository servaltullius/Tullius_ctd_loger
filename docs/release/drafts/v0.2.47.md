## 한눈에 보기
- 이번 버전은 **MO2 환경과 심볼 런타임 관련 오경고를 줄이는 후속 패치**입니다.
- `msdia140.dll`이 게임의 `SKSE\Plugins` 안에만 있어도 분석기가 실제로 찾아 사용할 수 있게 했습니다.
- MO2가 별도 경로에 설치된 경우에도 활성 profile의 `plugins.txt`를 더 잘 찾도록 보강했습니다.
- 플러그인 헤더를 읽지 못한 상태에서는 `ESP_FULL_SLOT_NEAR_LIMIT`를 확정적 경고처럼 띄우지 않도록 완화했습니다.

## 핵심 변경
- 분석기 stackwalk 런타임이 게임 설치 경로의 `Data\SKSE\Plugins\msdia140.dll`을 fallback으로 확인하고 필요 시 직접 로드합니다.
- helper/plugin scan이 `usvfs_x64.dll` 또는 `uvsfs64.dll`의 실제 모듈 경로를 따라 `ModOrganizer.ini`를 추적할 수 있게 했습니다.
- plugin scan 결과에 `slot_type_known`을 추가해, 헤더를 읽은 플러그인과 읽지 못한 플러그인을 구분합니다.

## 엔진 / 지원성
- `plugins.txt`를 찾지 못해 MO2 preflight가 제한되던 케이스를 줄이기 위해, 게임 폴더 인접 경로 외에 MO2 런처 위치 기반 fallback을 추가했습니다.
- 플러그인 헤더를 읽지 못한 경우에는 non-ESL 슬롯 한계 점검을 `approximate`로 낮춰, `0x28` 수준의 로드 오더를 `254 근접`처럼 잘못 해석하는 false positive를 줄였습니다.

## 빌드 / 운영
- 이번 변경은 helper preflight / plugin scanner / analysis engine runtime 테스트를 함께 확장해 회귀를 고정했습니다.
- 배포 zip은 `Tullius_ctd_loger_v0.2.47.zip` 형식을 사용합니다.

## 주의사항
- `game_version = 1.6.1179.0`에 대한 `address_db/skyrimse_functions.json` 함수 DB 지원은 이번 릴리즈 범위에 포함되지 않습니다.
- 이번 패치는 주로 `msdia`, `plugins.txt`, `plugin slot false positive` 같은 환경 탐지와 진단 품질 보정에 집중합니다.
