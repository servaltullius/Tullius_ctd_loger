# v0.2.42 — CTD/프리징 분석 개선 + 덤프 찾기/읽기 쉬움 개선

## 핵심 변경
- EXE 크래시처럼 원인이 애매한 경우에도, 예전보다 더 실용적인 후보를 보여주도록 개선했습니다.
- 프리징도 `데드락에 가까운지`, `로딩 멈춤에 가까운지`, `아직 애매한지`를 나눠서 보여주도록 바꿨습니다.
- 덤프를 찾는 시작 화면을 손봐서, MO2 `overwrite`나 설정한 출력 폴더 기준으로 더 쉽게 찾을 수 있게 했습니다.
- 후보 이름도 `보기 좋은 별칭`보다 `ESP/ESM`, `DLL 파일명`을 우선 보여주도록 바꿔서 실제 모드 파일을 찾기 쉬워졌습니다.

## WinUI / 사용성
- 첫 화면에서 최근 dump와 dump 출력 위치를 더 쉽게 확인할 수 있습니다.
- 분석 화면은 `무엇을 먼저 봐야 하는지`가 더 잘 보이도록 정리했습니다.
  - CrashLogger 기준
  - 근거 합의
  - 다음 조치
- 재수집된 dump라면, 왜 다시 수집됐는지와 어떤 방식으로 다시 수집됐는지를 UI에서 바로 볼 수 있습니다.
- 후보 이름이 `Faction Ranks` 같은 모호한 라벨보다 `FactionRanks.esp`, `paragon-perks.dll`처럼 실제 파일명 위주로 보입니다.

## 엔진 변경
- CTD 분석:
  - CrashLogger object ref
  - first-chance 문맥
  - stack / resource / history 신호
  를 함께 보고 후보를 정리하도록 강화했습니다.
- 프리징 분석:
  - WCT
  - blackbox 이벤트
  - snapshot 문맥
  - loading 상태
  를 함께 사용하도록 확장했습니다.
- 자동 재수집(recapture)도 상황에 따라 더 세밀하게 동작합니다.
  - 무조건 full dump로 올리지 않고
  - crash / freeze 상황에 맞는 richer profile을 먼저 시도합니다.

## 빌드 / 운영
- WSL에서 Windows 빌드를 더 안정적으로 돌릴 수 있게 했습니다.
- 릴리즈 노트와 prerelease 작성 흐름을 repo 안에 고정했습니다.
- 릴리즈 판단 기준을 GitHub CI보다 로컬 검증 기준으로 정리했습니다.
- 내부 구조 정리도 같이 진행해서, 이후 유지보수성이 좋아졌습니다.

## 주의사항
- 자동 dump 탐색은 일반 Windows `CrashDumps`가 아니라 툴리우스 출력 위치 기준입니다.
- freeze용 `PSS snapshot`은 아직 기본 활성화가 아니라 opt-in입니다.
- first-chance나 blackbox 정보는 보강 신호입니다. 단독으로 확정 원인을 의미하지는 않습니다.
