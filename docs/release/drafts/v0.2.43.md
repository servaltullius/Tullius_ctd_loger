# v0.2.43 — 기본 출력 폴더 정리 + MO2 overwrite 사용성 개선

## 핵심 변경
- `OutputDir`를 비워 둔 기본 설정에서도 dump/log가 `Tullius Ctd Logs` 하위 폴더로 모이도록 바꿨습니다.
- MO2 사용 시 `overwrite\\SKSE\\Plugins` 바로 밑이 빠르게 어지러워지는 문제를 기본 동작만으로 완화합니다.
- WinUI dump 자동 발견도 새 기본 폴더를 우선 따르도록 맞췄고, 기존 위치에 남아 있는 dump도 계속 찾습니다.

## WinUI / 사용성
- `OutputDir=` blank 상태에서 새로 생성되는 dump/log는 기본적으로 별도 하위 폴더에 모입니다.
- `OutputDir`를 직접 설정한 사용자는 기존처럼 절대경로나 상대경로를 그대로 사용할 수 있습니다.
- 경로 문자열에는 따옴표가 필요 없고, 상대경로는 helper 폴더 기준으로 해석됩니다.

## 엔진 변경
- Helper:
  - blank `OutputDir` 기본 해석을 전용 하위 폴더로 변경
  - 기존 명시 경로 계약은 유지
- WinUI:
  - blank `OutputDir`일 때 새 기본 root 우선 스캔
  - legacy 기본 root도 함께 스캔해 이전 dump를 계속 표시

## 빌드 / 운영
- 배포용 `SkyrimDiagHelper.ini` 주석을 새 기본 동작에 맞게 정리했습니다.
- README, 한글 문서, Nexus 설명을 현재 계약과 맞추었습니다.
- 실제 MO2 환경에서 `overwrite\\SKSE\\Plugins\\Tullius Ctd Logs` 생성까지 확인했습니다.

## 주의사항
- 이미 생성된 예전 dump는 기존 기본 위치에 남아 있을 수 있습니다.
- WinUI는 새 기본 위치를 우선 보되, legacy 위치도 함께 찾아서 전환기 동안 누락을 줄입니다.
- `OutputDir`를 직접 설정했다면 기존 설정이 우선이며, 이번 변경은 blank 기본값에만 적용됩니다.
